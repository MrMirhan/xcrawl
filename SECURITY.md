# Security Policy

## Supported Versions

XCrawl does not yet have tagged releases — only the latest `main` branch is supported with security fixes. If you're running an older commit or a fork, please update to the latest `main` before reporting, in case the issue is already fixed.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by emailing **security@xcrawl.space**. Do not open a public GitHub issue for security reports.

Please include:

- Steps to reproduce the issue
- The potential impact
- The affected version or commit hash

We aim to acknowledge reports within 5 business days. This is a small, self-hosted open-source project, so we can't promise a fixed resolution SLA, but we'll keep you updated as we work on a fix and credit you in the disclosure unless you'd prefer to stay anonymous.

## Scope

Vulnerabilities in XCrawl's own code are in scope for this policy. Vulnerabilities in third-party dependencies should generally be reported to the upstream project directly — though you're welcome to flag them to us too so we can track and update the affected dependency.
